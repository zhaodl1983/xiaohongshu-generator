一转眼就年底了，真的是年纪越大时间过得越快。2025年，这一年AI编程领域，可以说是高歌猛进的一年。

年初的时候，Cursor 还是一家独大。到现在，国内外科技大厂全部下场，AI编程工具遍地开花，截止到我发文，就自己用过的就有十几款，包括 Cursor 、 Claude Code 、 Kiro 、 Gemini CLI 等等。

今天这篇文章，我会从个人的视角，把 2025年 AI 编程领域的变化做一个盘点，然后给出我的实战建议。

四大核心变化
作为AI编程的使用者和分享者，我观察到几个明显的变化。

1. 界面和功能逐步趋同
各家工具在核心功能上越来越像：代码补全、对话生成、工具调用、多文件编辑，而差异主要在细节体验和模型选择上。

2. IDE和CLI走出独立路径
图形界面的 IDE ，适合大部分开发者；命令行的 CLI ，吸引了一批专业用户。两条路都有自己的受众，不是谁替代谁的关系。

3. 订阅套餐频繁调整
大模型调用成本居高不下，不少厂商都在调整订阅套餐。有的限制调用次数，有的提高价格，有的缩减功能。

4. 中转服务市场火爆
由于访问限制和费用问题，某宝/某鱼上的中转站和低价账号生意很火。这也从侧面说明了市场需求有多旺盛。

03 | 实战指南：如何选择你的AI工具组合
看到这里，很多朋友可能已经有点晕了。

接下来我站在自己的使用经验角度，给大家做一个简化版的推荐。


组合1：新手小白/纯兴趣玩家
用户画像：第一次接触AI编程，没有技术背景，纯粹感兴趣想玩玩。

推荐组合：Cursor + Gemini 2.5 Pro
推荐理由：Cursor 产品成熟度高，上手简单，支持市面上主流大模型，可以轻松切换，Gemini 2.5 Pro 便宜，Token 消耗相对少，对新手来说，功能足够用。

组合2：小应用开发者
用户画像：有一点技术背景，但不是写代码出身，想做点自己的小应用。

推荐组合（三选一）：
Cursor + Claude 3.5 Sonnet
Gemini CLI + Gemini 2.0 Flash
Cursor + GPT-4o
推荐理由：这三个组合都能稳定生成代码。在实际项目中输出结果会有不同，没有绝对好坏，完全看个人喜好。碰到一些难解的 bug 时，也可以切换工具组合试试，往往有奇效。

组合3：资深玩家/商业产品开发者
用户画像：有较好的技术背景、项目管理、代码工程经验，想做复杂应用，甚至打算上线赚钱。

推荐组合：Claude Code + Claude 3.5 Sonnet
推荐理由：Claude Code 作为 AI 编程命令行工具，受到广泛专业用户的喜爱。搭配 Claude 4.5 Sonnet 大模型，全球已有不少专业用户用这套组合进行商业产品开发和维护。



补充说明：以上推荐仅从我个人使用和认知出发，工具选择见仁见智，仅供参考。



04 | 开发模式之争：氛围开发 vs 规范开发 ？
2025年，在 AI 编程领域最火的一个词是：Vibe Coding（氛围开发）。

说人话：用人类的自然语言输入，AI 完成代码输出。

这个理念鼓励大家忽略代码本身，专注在语言描述上。甚至有人说：未来世界上只有一种编程语言，那就是英语。

Vibe Coding 在落地实际项目时，每个人的体验千差万别。但能确定的一点是：随着AI编程工具和大模型的进步，写代码这件事的门槛确实越来越低了。

我前段时间参加了一场 Cursor 北京线下活动，到场的人背景五花八门：有产品经理、有运营、有建筑设计师、有北交大的大学生，甚至还有高二的中学生。现场最直观的感受就是：人人都能用AI写代码，不分年龄、不分性别。

这也是 Vibe Coding 在 2025年全球爆火的一个缩影。

Spec Coding 的出现
Spec Coding 最早是今年7月在亚马逊的Kiro上线的。

在大家都在疯狂体验 Vibe Coding，但因为每个人的背景和能力不同，从预期到结果产生很大落差的时候，Spec Coding 提供了另一条路径。

Spec Coding（需求-设计-任务）
在AI IDE里给出了一条 Workflow，一套来自过去产研团队协作沉淀下来的最佳实践。让用户在使用AI编程时，先通过这条工作流，去澄清需求、确定技术架构和技术栈选择、将一个工程拆分为一个个可执行的模块和任务。

这迅速吸引了一波专业用户。其他AI IDE工具也很认同这个理念，纷纷更新迭代去支持。

我的体验：
Vibe Coding 更适合：新手小白入门，快速跑个原型，做纯前端的小应用。

不能期待通过 Vibe Coding 做出复杂应用。

我甚至见过有人平静地输入：请帮我做一个淘宝...

Spec Coding 更适合：有一定产品和技术背景的人，需要做相对复杂的应用，对需求理解、产品形态、UI设计、技术栈选择都有自己判断，注重代码的工程化和文档化管理。

我觉得未来这两种模式会并存很长时间。因为同样是AI编程，但不同人群、不同需求，会选择不同的使用模式。

05 | AI大模型的下一站？
AI大模型的发展在在追求什么？

简单一句话：用数学和算法模拟人脑的思考过程。

但人脑太复杂了。

目前所有的人工智能，都还只能在某一个特定领域，比如AI编程、文字生成图片/视频，通过大量的预训练去提高输出质量。

垂直化趋势
模型从通用大模型，在应用表现上逐步开始走向垂直领域，比如：

Claude 专门针对编程场景优化
Nano Banana 谷歌专门训练用来提升图片生成质量
Sora 专门用来生成视频
这个趋势在2026年可能会更明显。

06 | 终极问题：AI到底是替代人的，还是帮助人的？
现在互联网很发达，信息传递速度太快。很多新东西你还不了解，就被人一包装一吹嘘，你就容易产生焦虑。

天天琢磨：我们到底会不会被AI替代？

我的答案：不会
但AI在标准化、流程化、自动化方面的效率提升是不可忽视的。以前一些简单的、重复的、容易标准化的工作会被AI替代掉。

我们应该加强什么？
非标准状态下的能力：艺术审美、发现问题的能力、复杂问题抽象、关键问题决策。

这些东西，现在AI都干不了。

缓解焦虑的方式
看书，特别是关于人工智能底层原理的书。

虽然很枯燥、很难啃，但一旦你了解技术的底层原理后，很多东西你就祛魅了。

至少不会盲目崇拜，也不会盲目焦虑。

我强制要求自己，保持一周空出一个半天来，读一本书。

我更认同的观点
AI向前发展是一定的，因为人类社会从古至今都在不断提升效率。

我们应该把人工智能理解为"人类增强"或者"机器增强"。每个人未来都外挂一个人工智能大脑，帮我们处理重复性、标准化的事情，而人脑用来处理更复杂的推理、判断、决策。
